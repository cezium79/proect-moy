
# Документация структуры проекта "Охрана" (Для ИИ)

## 🛠 Стек технологий и архитектура
- **Язык**: Kotlin (100%)
- **UI Фреймворк**: Jetpack Compose
- **Локальное хранилище**: SharedPreferences (SharedPrefsManager)
- **Архитектура**: Плоская структура (Single Package / Feature-by-File)
- **Ключевой функционал**: Сканирование QR-кодов (ML Kit / Barcode), генерация отчетов об обходе.

## 📁 Назначение ключевых файлов (`app/src/main/java/com/example/ohrana/`)

### 🎨 Экраны интерфейса (Compose Screens)
- `PrivetScreen.kt` — Стартовый приветственный экран приложения.
- `AdministratorScreen.kt` — Панель управления для администратора системы.
- `EmployeeListScreen.kt` — Экран со списком сотрудников (вывод, добавление, менеджмент).
- `OhrannikScreen.kt` — Основное рабочее пространство охранника (интерфейс для дежурства).
- `OhrannikCabinetScreen.kt` — Личный кабинет/профиль сотрудника охраны.
- `SpisokOtchetov.kt` — Экран просмотра истории сгенерированных отчетов.

### ⚙ Логика, Утилиты и Данные
- `MainActivity.kt` — Главная Activity, координирующая навигацию между Compose-экранами.
- `Employee.kt` — Data-класс (модель данных), описывающий сотрудника.
- `QrHandler.kt` — Логика распознавания и парсинга QR-кодов.
- `SharedPrefsManager.kt` — Утилитный класс для сохранения сессий в SharedPreferences.
